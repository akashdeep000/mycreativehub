Switch password reset to a code-only, inline flow (no links).

Goal: After a user enters their email on “Forgot password”, keep them on the same screen and show inputs for the 6-digit code + new password. The reset email should contain only the code (no button/link).

1) Email content

Update the reset email template (in sendPasswordResetEmail or equivalent) to remove any link/button.

Subject: Your MyCreativeHub reset code: {{CODE}}

Body:

Your reset code is: {{CODE}}
Go back to the app and enter this code to set a new password.


Keep using the existing /auth/request-reset endpoint to generate & send the code.

2) Backend endpoints (reuse if already created)

POST /auth/request-reset
Input: { email }
Behavior: generate a 6-digit code, store it hashed with expiry (e.g., 10 min) and attempt counter, send email, return 200 always (do not leak if email exists).

POST /auth/confirm-reset
Input: { email, code, newPassword }
Behavior: verify code + expiry + attempts; on success, update password, invalidate code; return 200.

3) Frontend – 2-step, same-page UI

Keep route /forgot-password (no /reset-password route needed).

Step A (email):

Field: Email

Submit → call /auth/request-reset.

On success, do not navigate; switch the same view to Step B.

Show message: “If that email is registered, we’ve sent a 6-digit code. Check your inbox or junk.”

Step B (code + new password):

Fields: 6-digit Code (numeric, 6 boxes or one masked field), New password, Confirm password

Buttons: “Reset password”, “Resend code” (disabled with 30s cooldown)

Submit → POST /auth/confirm-reset with { emailFromStepA, code, newPassword }.

On success: show toast “Password updated” and redirect to Login.

On error:

400 invalid_code → “That code isn’t right. Please try again.”

400 expired_code → “Code expired. Tap Resend to get a new one.”

429 too_many_attempts → “Too many tries. Please wait a few minutes and request a new code.”

Include a “Back to Login” link.

4) Remove link-based paths

Remove any usage of /reset-password?... and any email template that inserts token URLs.

Ensure there’s no SPA fallback trying to handle /reset-password anymore.

5) Acceptance criteria

Submitting email keeps the user on the same screen and reveals code + new password inputs.

Email received contains only a 6-digit code (no button/link).

Entering email + code + new password successfully resets the password on both desktop and mobile.

“Resend code” works with cooldown and rate-limit.

No 404s or 500s during the flow.

QA: Try the whole flow on iPhone Safari and desktop Chrome.

Please implement, then Publish.
