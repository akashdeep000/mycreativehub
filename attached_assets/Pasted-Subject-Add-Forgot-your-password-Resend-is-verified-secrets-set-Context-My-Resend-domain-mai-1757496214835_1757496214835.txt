Subject: Add “Forgot your password?” (Resend is verified, secrets set)

Context
My Resend domain mail.mycreativehub.app is verified. In Production app secrets I’ve set:

EMAIL_PROVIDER=resend

EMAIL_API_KEY=<present>

EMAIL_FROM=no-reply@mail.mycreativehub.app

APP_BASE_URL=https://mycreativehub.app

Please wire up a standard reset flow using those.

A) UI (small additions)

Login page: add a “Forgot your password?” link beneath the password field → route to /forgot-password.

/forgot-password page: single email input + submit. On submit, call API and always show:
“If that email is registered, we’ve sent a reset link.”

/reset-password page: reads token from the URL; show new password + confirm, submit to API, then show success + link to login.

B) Backend endpoints

POST /auth/forgot-password

Input: { email: string }

Behavior:

Look up user silently (no user enumeration).

Generate a random token (32–48 bytes).

Store a hashed token (sha256) with: user_id, expires_at = now + 60 minutes, used = false.

Send an email via Resend (EMAIL_API_KEY, EMAIL_FROM) with link:
${APP_BASE_URL}/reset-password?token=<RAW_TOKEN>

Return 200 with a generic message.

POST /auth/reset-password

Input: { token: string, new_password: string }

Behavior:

sha256(rawToken) and find matching row where used = false and expires_at > now.

If valid, hash new_password (bcrypt 10–12), update the user, mark token used (or delete it), and revoke all other sessions for this user.

Return 200. On failure (expired/used/bad), return 400 with a friendly message.

(Optional) GET /auth/reset-password/validate?token=... for pre-checking the token when the page opens.

C) Data (suggested table)

Create table password_resets:

id uuid primary key,
user_id uuid not null,
hashed_token char(64) not null,      -- hex sha256
expires_at timestamptz not null,
used_at timestamptz,
created_at timestamptz default now(),
unique (hashed_token)


Indexes: (user_id), (hashed_token).

D) Email (Resend)

Subject: Reset your MyCreativeHub password

HTML (feel free to style it):

<p>Hi there,</p>
<p>Click the button below to reset your password. This link expires in 60 minutes.</p>
<p><a href="{{resetUrl}}" 
   style="display:inline-block;padding:12px 18px;background:#e75252;color:#fff;
          text-decoration:none;border-radius:8px;">
   Reset Password
</a></p>
<p>If the button doesn’t work, copy and paste this link:</p>
<p style="word-break:break-all">{{resetUrl}}</p>
<p>If you didn’t request this, you can ignore this email.</p>


Build resetUrl as: ${APP_BASE_URL}/reset-password?token=${RAW_TOKEN}

Also send a plain-text version.

E) Security must-haves

Hash tokens at rest (sha256). Compare in constant time.

Single-use tokens. Expire in 60 minutes.

Rate-limit /auth/forgot-password (e.g., 5/hour per IP/email).

Return the same success message whether or not the email exists.

After password change, invalidate existing sessions (logout on other devices).

CORS/cookies set appropriately for mycreativehub.app.

F) Acceptance criteria

Requesting a reset always returns success UI; Resend Logs show an email sent to valid users.

The link opens the reset page, accepts a new password, and logs me in (or takes me to login).

Reusing the link, using an altered token, or using an expired token is rejected with a friendly error.

Existing sessions are revoked after the change.
