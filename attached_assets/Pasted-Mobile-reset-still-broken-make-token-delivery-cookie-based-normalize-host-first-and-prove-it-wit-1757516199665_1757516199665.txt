Mobile reset still broken — make token delivery cookie-based, normalize host first, and prove it with headers

Mobile links work inconsistently because the token in the query string gets lost or the cookie isn’t sticking after redirects. Please implement the robust flow below and show the headers to prove it’s correct.

1) Normalize scheme/host before any reset routes

Make sure every request hits the same host over HTTPS and keeps the full URL (no path/query loss) before the reset flow:

app.set('trust proxy', 1);
app.use((req, res, next) => {
  // Force HTTPS
  if (req.headers['x-forwarded-proto'] !== 'https') {
    return res.redirect(301, `https://${req.headers.host}${req.originalUrl}`);
  }
  // Force apex (no www), preserving full path + query
  if (req.headers.host === 'www.mycreativehub.app') {
    return res.redirect(301, `https://mycreativehub.app${req.originalUrl}`);
  }
  next();
});

2) Cookie-handoff endpoint (no query strings in the final page)

Set the reset token in a first-party cookie and then redirect to a clean page:

// GET /rp/:token  (short link used in emails)
app.get('/rp/:token', (req, res) => {
  const token = req.params.token;
  if (!token) return res.status(400).send('Missing token');

  res.cookie('reset_token', token, {
    httpOnly: true,
    secure: true,
    sameSite: 'lax',
    domain: '.mycreativehub.app', // <-- IMPORTANT: covers apex and subdomains
    path: '/',
    maxAge: 1000 * 60 * 15
  });

  return res.redirect(302, '/reset-password'); // no token in the URL
});

Server must be ESM-safe (no require('path') errors):

After API routes, serve the SPA for non-API GETs without redirecting:

Error handler: JSON for /api/*, SPA shell for everything else.

5) Backend reads cookie on reset submit & clears it
// POST /api/auth/reset-password  (body: { password })
app.post('/api/auth/reset-password', async (req, res) => {
  const token = req.cookies.reset_token;
  if (!token) return res.status(400).json({ message: 'Missing reset token' });
  // ...validate token, set new password...
  res.clearCookie('reset_token', { domain: '.mycreativehub.app', path: '/', httpOnly: true, secure: true, sameSite: 'lax' });
  return res.json({ ok: true });
});

If any redirect occurs after /rp/:token to a different host (e.g., www → apex) or if the cookie lacks Domain=.mycreativehub.app, the cookie will be lost on mobile. Please verify the normalize-host middleware runs before /rp/:token and that the cookie Domain is set exactly as above.
